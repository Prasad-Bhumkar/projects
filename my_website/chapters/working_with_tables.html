Working with Tables

Introduction to Tables

Tables are used to display data in a structured format, allowing users to easily read and understand the information. In HTML, tables are created using the <table> tag. Tables are commonly used to display data in a tabular format, making it easier to compare and analyze information.

Some common use cases for tables include:

  • Displaying data in a spreadsheet-like format
  • Comparing features and prices of different products
  • Showing schedules and timetables
  • Presenting statistical data in a clear and concise manner

Table Attributes and Styles

Tables can be customized using various attributes and styles. Here are some examples:

Border Styles

Use the border attribute to set the border style:

<table border="1">
  <tr>
    <th>Header 1</th>
    <th>Header 2</th>
  </tr>
  <tr>
    <td>Data 1</td>
    <td>Data 2</td>
  </tr>
</table>

Output:

Header 1 Header 2
Data 1 Data 2

Cell Padding and Spacing

Use the cellpadding and cellspacing attributes to adjust the padding and spacing between cells:

<table cellpadding="10" cellspacing="5">
  <tr>
    <th>Header 1</th>
    <th>Header 2</th>
  </tr>
  <tr>
    <td>Data 1</td>
    <td>Data 2</td>
  </tr>
</table>

Output:

Header 1 Header 2
Data 1 Data 2

Best Practices for Tables

Here are some best practices to keep in mind when working with tables:

  • Use tables only for tabular data, not for layout purposes.
  • Use clear and concise header text to help users understand the data.
  • Avoid using too many columns or rows, as it can make the table difficult to read.
  • Use CSS to style your tables and make them visually appealing.
  • Make sure your tables are accessible by using proper semantic markup and providing alternative text for images.
